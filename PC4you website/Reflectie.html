Reflectie van de opdracht

Op deze pagina wil ik het even hebben over het leer proces.
Ik zal hier bij een aantal paginas, soms meerdere tegelijk, het hebben over wat de obstakels waren
en wat ik ervan heb geleerd. dus hierbij de reflectie van dit PO.

Home Page

De Home Page was het eerste grote obstakel, omdat ik toen ook vanaf niks de css moest maken en ik voor een van de eerste keren blokken moest toepassen. Maar toen het eindelijk wel lukte had ik wel de mooiste pagina op de website. Eén van de dingen die ik leerde was de functie button maar nog blangrijker de functie border-radius, die ik regelmatig daarna gebruite bij onder anderen blokken en die zie je dus als je goed kijkt veel terug komen, en ik maakte fouten tijdens het begin die ik daarna nooit meer maakte, uit eindelijk had ik de echte basis daarna voledig onder controle op een manier dat eerder niet het geval was.

Textpagina's (About Us, Bestellingen, Credits)

hier is een voorbeel van waar ik achteraf nog de functie border-radius gebruikte. Hier had ik vooral kuren met het middelste blok, qua positionering en formaat. Voor de rest was de uitdaging om een balans te vinden tussen simpel, maar ook mooi en niet saai.

Contact

Contact ging vrij frictieloos vooral, omdat ik bij de andere textpagina's al met de layout gewerkt had waardoor het gewoon werkte, en ik verder niks onbekends deed. Hierbij heb ik wel iets meer geleerd over Hyperlinks, vooral hoe je een Hyperlink kan laten openen in een nieuw tab. En als u op de icons klikt van de social media wordt u doorgestuurd naar echte social media pagina's die ik voor dit project maakte.

Custom Desktop/Desktop Builder

Hier bij het formulier was de aller grootste uitdaging, want eigenlijk waren alle mogelijke aanpakken om een formulier geïntegreerd in de site te hebben niet mogelijk. Meeste aanpakken hadden een webserver nodig of op z'n minst kennis van een ander programeer taal. Er was wel tijdens de les een oplossing bedacht, maar daar konden geen checkboxes en dat vernietigde nou juist het hele nut van dat formulier in de website bouwen. Ik kwam dus uit eindelijk tot de oplossing om JavaScript te gebruiken, nu MOET ik hier duidelijk maken dat ik deze code niet zelf heb geschreven, maar heb gekopieerd, omdat ik nul ervaring heb in JavaScript en er niet een alternatief was. Wat ik wel heb gedaan is, ik heb hier wel zelf alle HTML geschreven en geprobeerd in de code zo duidelijkmogelijk te maken welke gedeeltes ik zelf heb gedaan en welke gedeeltes gekopieerd. Ik heb toch ook kleine geeeltes in de JavaScript aangepast, ik kwam er namelijk achter dat het niet zoveel anders is dan Python waar ik begin kennis in heb. Ik heb hier ook geleerd over HTML forms en over de conecties die die forms hebben met andere code of met een action.
Kortom: hoewel ik niet alles volledig zelf heb gedaan op deze pagina heb ik hier wel het meest van geleerd.

Conclusie

Hoewel dit project veel van mij heeft geëist, ben ik heel tevreden met het eindresultaat en ik vonde het ook daadwerkelijk leuk om HyperText te leren en heb ik ergens zin om er zelf nog een website te maken.
qua cijfer ben ik te bang om mezelf heel hoog te geven, maar zoals ik al zei ben ik heel erg tevreden met het product, ook heb ik het idee dat de website ook het beste is wat ik hem kan maken, voor verdere verbetering zou ik zo ver buiten de eisen moeten treden en zou ik zo vanaf begin tot eind alleen, maar mijn eigen pad moeten graven, wat ik voor mijn idee al een klein beetje heb gedaan.
Dus het cijfer dat ik mijzelf zou geven zou zijn een 8.5.
